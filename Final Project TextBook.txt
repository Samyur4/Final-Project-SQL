SQL ტერმინები და შეზღუდვები (Constraints)ეს არის წესები, რომლებიც მონაცემთა ბაზაში მონაცემების სისწორეს უზრუნველყოფს:UNIQUE: უზრუნველყოფს, რომ სვეტში ყველა მნიშვნელობა იყოს განსხვავებული. მაგალითად, ორ წევრს არ შეიძლება ჰქონდეს ერთი და იგივე Email.PRIMARY KEY (PK): უნიკალური იდენტიფიკატორი თითოეული ჩანაწერისთვის (მაგ. BookID). ის არასოდეს შეიძლება იყოს ცარიელი.FOREIGN KEY (FK): "გარე გასაღები", რომელიც აკავშირებს ერთ ცხრილს მეორესთან. ის მიუთითებს სხვა ცხრილის Primary Key-ზე.IDENTITY(1,1): ავტომატური დანომვრა. იწყებს 1-დან და ყოველ ახალ ჩანაწერზე ზრდის 1-ით.NOT NULL: ნიშნავს, რომ ველი სავალდებულოა და არ შეიძლება დარჩეს ცარიელი.CHECK: ლოგიკური შეზღუდვა. მაგალითად, CHECK (Amount >= 0) კრძალავს ჯარიმის უარყოფითი რაოდენობის შეყვანას.DEFAULT: მნიშვნელობა, რომელიც ავტომატურად ჩაიწერება, თუ ჩვენ სხვას არ მივუთითებთ (მაგ. IsPaid თავიდან ყოველთვის არის 0/False).2. ურთიერთობები (Relationships)თქვენს დიაგრამაზე ხაზები აჩვენებს, როგორ უკავშირდებიან ცხრილები ერთმანეთს:One-to-Many (ერთი ბევრთან $1:\infty$): ყველაზე გავრცელებული კავშირი.მაგალითი: ერთ გამომცემლობას (Publisher) შეუძლია გამოსცეს ბევრი წიგნი (Books).Many-to-Many (ბევრი ბევრთან $M:N$): როდესაც ბევრ ჩანაწერს ერთ ცხრილში შეესაბამება ბევრი ჩანაწერი მეორეში.Junction Table (შემაერთებელი ცხრილი): SQL-ში პირდაპირი "ბევრი ბევრთან" კავშირი არ არსებობს. ამისთვის გამოიყენეთ BookAuthors, რომელიც აკავშირებს წიგნებსა და ავტორებს.Referential Integrity (რეფერენციული მთლიანობა): წესი, რომელიც კრძალავს წიგნის წაშლას, თუ მასზე აქტიური სესხია (Loan) მიბმული.3. Entity Framework Core (ხიდი კოდსა და ბაზას შორის)ეს არის ტექნოლოგია, რომლითაც C# "ელაპარაკება" SQL სერვერს:Scaffolding (Database-First): პროცესი, როდესაც EF Core კითხულობს თქვენს SQL ბაზას და ავტომატურად ქმნის C# კლასებს (Models).DbContext: მთავარი "მმართველი" კლასი (LibraryDbContext). ის ინახავს კავშირს სერვერთან და აკონტროლებს მონაცემებში ცვლილებებს.DbSet: ცხრილის შესაბამისი კოლექცია C#-ში (მაგ. DbSet<Book> არის Books ცხრილი კოდში).LINQ: ენა, რომლითაც ვწერთ მოთხოვნებს კოდში (მაგ. .Where, .OrderBy).Migrations (მიგრაციები): გზა, რომლითაც კოდში შეტანილი ცვლილებები (მაგ. Seed Data) გადადის SQL ბაზაში.4. CRUD ოპერაციები (მოქმედებები)ეს არის ის, რასაც თქვენი პროგრამა რეალურად აკეთებს:Create (შექმნა): ახალი ობიექტის დამატება .Add() მეთოდით.Read (წაკითხვა): მონაცემების ამოღება ბაზიდან .ToList() ან .FirstOrDefault() მეთოდებით.Update (განახლება): არსებული მონაცემის შეცვლა (მაგ. გვარის შეცვლა).Delete (წაშლა): ჩანაწერის წაშლა .Remove() მეთოდით.db.SaveChanges(): ყველაზე მნიშვნელოვანი ბრძანება. სანამ ამას არ დაწერთ, SQL ბაზაში არაფერი შეიცვლება.5. აგრეგაცია და ნავიგაციაAggregation (აგრეგაცია): გამოთვლები მონაცემთა ჯგუფზე: COUNT (რამდენი?), SUM (ჯამში რამდენი?).Navigation Property (ნავიგაცია): C#-ში Join-ის ნაცვლად ვიყენებთ მარტივ წერტილს: book.Genre.Name. EF Core თავად პოულობს შესაბამის ჟანრს სხვა ცხრილში.
